# スケジュール管理アプリ

## 概要
プログラミングスクールの実践課題として、Ruby on Railsの基礎を学ぶ目的で作成した  
個人向けのスケジュール管理アプリです。  
MVC構成を意識しながら、予定の登録・閲覧・編集・削除といったCRUD機能を実装しました。  
また、入力ミスを防ぐためのバリデーションを設定し、実用性を意識して開発しました。

## 作成した目的
本アプリは、プログラミングスクールの実践課題として、  
RailsにおけるCRUD処理の流れや、モデル・ビュー・コントローラそれぞれの役割を  
理解することを目的として作成しました。  
単に動くものを作るだけでなく、ユーザーが迷わず操作できる画面や入力制限を意識しています。

## 使用技術
- Ruby  
- Ruby on Rails 7.2.2.1  
- HTML / CSS  
- Git / GitHub  
- GitHub Codespaces（Windows環境での開発）

## 機能一覧
- スケジュール一覧表示  
- スケジュール新規作成  
- スケジュール詳細表示  
- スケジュール編集・更新  
- スケジュール削除  

## バリデーション
- タイトル：必須、20文字以内  
- 開始日：必須  
- 終了日：必須、開始日以降の日付  
- メモ：500文字以内  

## 工夫した点
- バリデーションエラー発生時にエラーメッセージを表示し、  
  ユーザーがどの項目を修正すればよいか一目で分かるようにしました。
- 日付の前後関係など、実際の利用を想定した入力制限を設けました。
- スクール課題ではありますが、Railsの標準的な書き方を意識し、  
  可読性を重視して実装しました。

## 苦労した点
Windows環境でRailsのローカル環境構築がうまくいかず、アプリを起動できない問題に直面しました。  
そのためGitHub Codespacesを利用して開発環境を整え、エラー内容を一つずつ調べながら  
課題に取り組みました。  

また、実装手順を誤り一度アプリを最初から作り直すことになりましたが、  
その経験から「設計や手順を整理してから実装することの重要性」を学びました。

## 振り返り・今後の改善点
プログラミングスクールの課題を通して、  
Railsを用いたWebアプリ開発の基本的な流れを理解できました。  
今後は以下のような機能追加を行い、理解をさらに深めたいと考えています。

- ユーザー認証機能の追加（ユーザーごとのスケジュール管理）
- UI・デザインの改善
- 検索機能や並び替え機能の追加
